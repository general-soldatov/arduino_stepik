**Вова**: Круто! Только нам придётся постоянно держать кнопку, чтобы свет работал???

**Маша**: Нет, конечно! Можно сделать такую конструкцию: создаём логическую переменную, в которой храним предыдущее состояние, это как флажок в программе ты ставишь в тестах на выбор нескольких правильных ответов. По умолчанию значение будет ложным, т.е. равно нулю:

```cpp
bool onLED = false;
```
Затем мы проверяем, в функции `loop` проверяем, нажата ли кнопка. Если кнопка нажата, то далее проверяем условие, ложна ли переменная `onLED`, и, если это так, то включаем свет, а переменной `onLED` присваиваем 1. Иначе - делаем наоборот. Это называется вложенная условная конструкция.
```cpp
if (digitalRead(10) == LOW) {
  if (onLED == false) {
    digitalWrite(9, HIGH);
    onLED = 1;
  } else {
    digitalWrite(9, LOW);
    onLED = 0;
}
```
**Вова**: Тогда я думаю, что у нас всё получится! А этот код не будет мешать поворотникам?

**Маша**: Нет, у нас же идёт опрос кнопок не связанный между собой! Только, конечно, когда поворотник включён, то запрос состояния кнопок мы сделать не сможем, поскольку микроконтроллер будет выполнять цикл.
